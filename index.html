The Mathematical Blueprint: Equations of a Neural Network

Just like an architect uses blueprints to translate a vision into a physical building, we'll use mathematical equations to describe our neural network. Don't worry; it's not as daunting as it sounds! We'll break it down step by step.

Continue

Let's start with a single perceptron in your network. Remember its equation?

\[ a = \varphi(\sum(w_i * x_i) + b) \]

where:

  • a is the neuron's output
  • Ï† is the activation function
  • wi are the weights
  • xi are the inputs
  • b is the bias

Now, in a multi-layer network, the output of one layer becomes the input for the next. So, for each layer, we'll need to calculate the weighted sum and apply the activation function for each node. Let's use a simple example.

You'll need to write similar equations for each node in each hidden layer and the output layer. Notice how the output of one layer becomes the input for the next, creating a chain of calculations. That's the essence of a feedforward network!

Assume your first hidden layer has 4 nodes, and your input layer has 3 nodes. For each node in the hidden layer, you'll have an equation like this:

\[ h_1 = \phi(w_{11}x_1 + w_{12}x_2 + w_{13}x_3 + b_1) \]

Where:

  • h1 is the output of the first node in the hidden layer.
  • w11, w12, w13 are the weights connecting the input layer to the first hidden node.
  • x1, x2, x3 are the inputs from the input layer
  • b1 is the bias of the first hidden node.

Stop and Think

Looking at the equation for h1, can you see how changing a single weight (e.g., w11) would affect the output of the network?

Changing w11 would directly affect h1, which then becomes an input for the next layer, influencing all subsequent calculations. Even small weight changes can have a ripple effect throughout the network!
